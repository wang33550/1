# v0.1.2

## 概要

`v0.1.2` 的重点是把项目继续往“真实用户可用”推进两步：

1. 用户第一次进入某个工作区时，`claude` / `codex` 启动链路会自动补齐缺失配置与 hooks  
2. 新增 `trr export-feedback`，可以把真实会话导出成脱敏反馈包，便于后续收集失败样本和迭代

## 主要变化

- 新增 `trr export-feedback`
- 默认导出当前工作区最新会话
- 默认对工作区路径、家目录路径、常见 API key / Bearer token 做脱敏
- 反馈包中包含：
  - 最新 checkpoint
  - 当前恢复包
  - 关键时间线
  - guard 结果
  - 最近 artifacts
  - 人工标签和备注
- `wrap claude` / `wrap codex` 首次启动时自动补齐：
  - `trr.config.json`
  - Claude 工作区 hooks
  - Codex hooks
- README 与 CLI 文档切换为中文主说明

## 实际效果

现在更接近用户真正想要的使用方式：

```bash
trr setup
```

之后：

```bash
cd /你的项目
claude
```

或：

```bash
cd /你的项目
codex
```

当你遇到真实恢复失败案例时，可以直接执行：

```bash
trr export-feedback --last
```

把脱敏后的反馈包发回仓库维护者，而不必手工整理整段对话。
